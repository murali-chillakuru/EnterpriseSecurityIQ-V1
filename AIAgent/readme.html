Pipeline Overview
sequenceDiagram participant U as User / CLI participant A as agent.py participant O as orchestrator.py participant C as Collectors (68) participant E as Evaluators (10) participant R as Reports (29) U->>A: run_assessment(scope, frameworks) A->>O: run_full_assessment(creds, config) par Azure Collectors O->>C: 50 Azure collectors (batches of 8) C-->>O: Azure evidence and Entra Collectors O->>C: 18 Entra collectors (batches of 6) C-->>O: Entra evidence end O->>E: evaluate_all(evidence, frameworks) E-->>O: FindingRecord[] + scores O->>R: generate_reports() R-->>O: HTML, JSON, MD, Excel, OSCAL, SARIF, PDF O-->>A: AssessmentSummary A-->>U: Formatted response
Three-phase pipeline:
  1. Collect — Azure and Entra groups run concurrently via asyncio.gather(), batched to avoid throttling. Checkpoint saved after each batch for resume.
  2. Evaluate — 120 functions across 10 domains. Cross-domain fallback for unassessed controls. Severity-weighted scoring.
  3. Report — Per-framework HTML/JSON in subfolders. Master report, executive dashboard, optional PDF conversion via Playwright.

Quick Start

  1. Install Dependencies

    cd AIAgent
pip install -r requirements.txt
pip install debugpy agent-dev-cli --pre

  2. Configure Environment

    cp .env.template .env
# Edit .env with your Foundry project endpoint and model deployment name

    Required: FOUNDRY_PROJECT_ENDPOINT, FOUNDRY_MODEL_DEPLOYMENT_NAME (default: gpt-4.1), optional AZURE_TENANT_ID.

  3. Authenticate

    az login

    Uses unified DefaultAzureCredential for both ARM and Graph — no browser popup required.

  4. Run Locally

    python main.py
# Agent starts on http://localhost:8088

  5. Debug with AI Toolkit

    In VS Code, press F5 → select Debug EnterpriseSecurityIQ Agent (HTTP Server). Launches with agentdev instrumentation and opens the AI Toolkit Agent Inspector.

☁️

Deploy to Foundry

Protocol: responses v1 on port 8088. Container image, published to ACR, then referenced in Foundry hosted agent creation.
# Build & push container
docker build --platform linux/amd64 -t enterprisesecurityiq-agent .
docker tag enterprisesecurityiq-agent <your-acr>.azurecr.io/enterprisesecurityiq-agent:latest
docker push <your-acr>.azurecr.io/enterprisesecurityiq-agent:latest

SDK versions: agent-framework-azure-ai==1.0.0rc3, agent-framework-core==1.0.0rc3, msgraph-sdk==1.12.0, msgraph-beta-sdk==1.12.0
