Overview

The Data Security Engine (data_security_engine.py) is a 9,904-line, purely functional assessment engine that evaluates data-layer security posture across Azure and Microsoft 365 tenants. It collects evidence from Azure Resource Graph, ARM sub-resource APIs, and Microsoft Graph, then runs 43 category analyzers comprising approximately 160 individual check functions. Every finding is backed by concrete evidence — resource IDs, configuration snippets, and API responses — indexed by 40+ evidence types for O(1) lookup. The engine maps all findings to 8 compliance frameworks and produces structured output across 7 report formats.

Design Philosophy: Evidence-first, purely functional (no classes), deterministic output. Every check function receives an immutable evidence index and returns findings — no side effects, no shared mutable state. This makes the engine trivially testable and CI/CD friendly.
⚡

Assessment Pipeline

The data security assessment follows a collect → enrich → index → analyze → score → report pipeline. Evidence is collected from three sources (Azure Resource Graph, ARM APIs, Microsoft Graph), indexed by type for constant-time lookup, then fed to 43 category analyzers. Compliance mappings are enriched post-analysis before scoring and report generation.

End-to-End Assessment Pipeline
sequenceDiagram participant U as User / CI Pipeline participant R as run_data_security.py participant E as DataSecurityEngine participant ARG as Azure Resource Graph participant ARM as Azure ARM APIs participant G as Microsoft Graph participant S as Scoring Engine participant RP as Report Generators U->>R: python run_data_security.py --tenant T R->>E: run_data_security_assessment(creds) E->>ARG: _ds_lightweight_collect() ARG-->>E: Resource configurations E->>ARM: _ds_arm_enrich() — sub-resource details ARM-->>E: TDE, auditing, keys, firewall … E->>G: M365 DLP, labels, SharePoint G-->>E: M365 evidence E->>E: Index evidence by type → O(1) lookup E->>E: Run 43 category analyzers (~160 checks) E->>E: enrich_compliance_mapping() — 8 frameworks E->>S: compute_data_security_scores() S-->>E: Scores + severity distribution E-->>R: Full assessment result R->>RP: HTML + Excel + JSON + PDF + scripts RP-->>U: Output files written
  1. Collect: _ds_lightweight_collect() queries Azure Resource Graph for storage accounts, SQL servers, key vaults, Cosmos DB, networking, and more.
  2. Enrich: _ds_arm_enrich() calls ARM sub-resource APIs for TDE status, auditing policies, encryption keys, firewall rules, and advanced threat protection.
  3. M365: Microsoft Graph queries retrieve DLP policies, sensitivity labels, SharePoint site configurations, and retention policies.
  4. Index: Evidence is organized by type (e.g., azure-storage-security) into a dictionary for O(1) lookup during analysis.
  5. Analyze: 43 category analyzers run ~200 individual checks, each producing typed findings with severity, remediation, and affected resources.
  6. Compliance: enrich_compliance_mapping() maps findings to 8 frameworks (CIS, PCI-DSS, HIPAA, NIST, ISO 27001, SOC 2, CSF, MCSB).
  7. Score: compute_data_security_scores() computes per-category and overall weighted scores.
  8. Report: Generates HTML, Excel, JSON, PDF, executive brief, and remediation scripts.

Scoring Model

The scoring engine converts raw findings into actionable risk scores. Each finding's severity maps to a numerical weight. Per-category scores are computed first, then aggregated into an overall weighted average based on finding counts.

Severity Weights

SeverityWeightBadge
Critical10Critical
High7.5High
Medium5Medium
Low2.5Low
Info1Info

Score Calculation

Per-category score: min(100, sum(finding_weights) × 5)
Overall score: Weighted average of category scores, weighted by finding count per category.

Risk Levels

Score RangeRisk LevelBadge
≥ 75CriticalCritical
≥ 50HighHigh
≥ 25MediumMedium
< 25LowLow

Score Output Structure

FieldDescription

SeverityDistributionCount of findings by severity level
TopFindingsHighest-impact findings ranked by weight
RemediationImpactProjected score improvement per remediation action
TrendAnalysisScore comparison against previous assessment runs

Compliance Framework Mappings

Every finding produced by the engine is enriched with compliance mappings via enrich_compliance_mapping(). The mapping data is maintained in structured JSON definitions that associate each check with specific controls from 8 industry and regulatory frameworks. This enables compliance-filtered reporting and gap analysis out of the box.

Enrichment Process: After all 43 category analyzers complete, enrich_compliance_mapping() iterates through every finding, looks up the associated category key and subcategory, and attaches an array of framework control references. CLI Usage

The data security assessment is driven by run_data_security.py. It supports full runs, evidence reuse, category filtering, trend comparison, suppressions, and CI/CD gating. 

# Full assessment
python run_data_security.py --tenant YOUR_TENANT_ID

# Reuse previously-collected evidence
python run_data_security.py --tenant YOUR_TENANT_ID --evidence-dir output/20260410_102337_PM

# Filter to specific categories
python run_data_security.py --tenant YOUR_TENANT_ID --categories storage,keyvault,database

# Trend comparison against a previous run
python run_data_security.py --tenant YOUR_TENANT_ID --compare output/20260409_083000_AM

# Apply finding suppressions
python run_data_security.py --tenant YOUR_TENANT_ID --suppressions config/suppressions.json

# CI/CD gate — exit non-zero if critical findings exist
python run_data_security.py --tenant YOUR_TENANT_ID --fail-on critical

# List available categories
python run_data_security.py --list-categories

# Specify output format
python run_data_security.py --tenant YOUR_TENANT_ID --format json,html,excel

Arguments Reference

ArgumentRequiredDescription

--evidence-dirNoPath to previously-collected evidence for offline analysis
