Quick Start

Get up and running in four steps. All operations use DefaultAzureCredential — no secrets stored in config files.

1

Install

Clone the repository and install dependencies:

git clone https://github.com/your-org/EnterpriseSecurityIQ.git
cd EnterpriseSecurityIQ/AIAgent
pip install -r requirements.txt
2

Authenticate

Sign in with Azure CLI — the agent uses DefaultAzureCredential:

az login
az account set --subscription "your-subscription-id"
3

Configure

Edit the config file at config/enterprisesecurityiq.config.json with your subscription ID, tenant ID, and desired frameworks.

4

Run Assessment

Execute a full compliance assessment:

cd AIAgent
python run_assessment.py

Reports are generated in the output/ directory with timestamped folders.

Safety Guarantees

🛡️

100% Read-Only Operations

PostureIQ performs zero write, update, or delete operations against your Azure or Entra ID environment. Every API call is a GET or list operation.

  • No resource modifications — ever
  • No policy changes or assignments
  • No user or group modifications
  • No role assignment changes
  • No network or firewall changes
  • No Key Vault secret access (metadata only)
  • No data-plane writes to storage/databases
  • Preflight permission check before collection
🔏

Report Integrity

Every generated report includes a SHA-256 cryptographic hash for tamper detection. The hash covers the complete report content and is embedded in the output JSON metadata.

🔐 SHA-256 Verification — Each report output includes a content_hash field computed at generation time. Re-compute the hash on the report body to verify integrity: 
python -c "import hashlib,json; d=json.load(open('report.json')); print(hashlib.sha256(json.dumps(d['results'],sort_keys=True).encode()).hexdigest())"
📋 Audit Trail — Every assessment run records: timestamp, subscription ID, tenant ID, collector versions, framework versions, and the authenticated principal. This metadata is included in all output formats.
