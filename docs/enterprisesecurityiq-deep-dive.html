Part 1: Why Does Any of This Matter? L100

Imagine your company stores customer data — credit card numbers, medical records, personal email addresses — in the cloud. Now imagine a hacker steals all of it because a door was left open. That's not hypothetical. It happens every day. Compliance is the discipline of making sure those doors are locked, monitored, and auditable.

What Is "The Cloud"?

Instead of buying physical servers and putting them in your office, you rent computing power from companies like Microsoft (Azure), Amazon (AWS), or Google (GCP). These providers manage data centers around the world. When you "move to the cloud," your applications, databases, and files live in those data centers instead of under your desk.

Think of cloud computing like renting an apartment instead of buying a house. The landlord (Microsoft) maintains the building, but you're responsible for locking your own door and not leaving the stove on.

What Is Compliance?

Compliance means proving that your organization follows specific security rules. These rules come from:

  • Government regulations — FedRAMP (US Federal), HIPAA (healthcare), PCI-DSS (credit cards)
  • International standards — ISO 27001 (global security standard), NIST 800-53 (US government)
  • Industry benchmarks — CIS Benchmarks (community best practices), MCSB (Microsoft's own)

If your company sells to the US government, you must be FedRAMP authorized. If you process credit cards, you must comply with PCI-DSS. Failure means fines, lawsuits, lost contracts, and reputational damage.

What Is a "Control"?

A control is a specific security requirement. Think of it as a checkbox on a very long to-do list. Examples:

🔑

AC-2: Account Management

Every user account must be tracked, reviewed regularly, and disabled when people leave the company.

🔒

SC-28: Protection of Information at Rest

All stored data must be encrypted — if someone steals the hard drive, they see only random noise.

📊

AU-2: Audit Events

The system must log who did what and when — so investigators can trace an incident back to its source.

A typical compliance framework has 20–300+ controls. Manually checking each one across hundreds of cloud resources is a months-long project. That's the problem PostureIQ solves — it automates this entire process.

Part 2: Microsoft Azure & Entra ID Explained L100

What Is Microsoft Azure?

Azure is Microsoft's cloud platform offering 200+ services. PostureIQ focuses on the security-critical ones:

🖥️ Compute

Virtual Machines (VMs) — full servers you control. App Services — managed web hosting. Azure Kubernetes Service (AKS) — container orchestration. Azure Functions — serverless event-driven code.

🌐 Networking

Virtual Networks (VNets) — private networks in the cloud. Network Security Groups (NSGs) — firewalls on steroids. Azure Firewall — centralized traffic filtering. Route Tables — control where traffic flows.

💾 Storage & Databases

Storage Accounts — blobs, files, queues. Azure SQL — managed SQL databases. Key Vault — secrets, certificates, and encryption keys. Cosmos DB — globally distributed NoSQL.

🛡️ Security

Microsoft Defender for Cloud — threat detection and security posture. Azure Policy — guardrails that enforce rules automatically. RBAC — Role-Based Access Control (who can do what).

Control Plane vs. Data Plane

This distinction is crucial for understanding PostureIQ:

Control Plane (Management)

Operations that manage resources: create, delete, configure, list. Uses Azure Resource Manager (ARM) APIs. Example: "List all storage accounts" or "Get Key Vault configuration."

PostureIQ uses this for 39 of 44 Azure collectors.

Data Plane (Content)

Operations that access data inside resources: read a blob, query a database, list secrets in Key Vault. Uses service-specific endpoints.

PostureIQ uses this for 5 data-plane collectors — ACR, AKS in-cluster, APIM, Cosmos DB, and Storage data plane (metadata only — never reads data values).

What Is Microsoft Entra ID?

Formerly called "Azure Active Directory" (Azure AD), Entra ID is Microsoft's cloud identity service. It answers three questions:

  1. Who are you? (Authentication — usernames, passwords, MFA)
  2. What can you do? (Authorization — roles, permissions, Conditional Access)
  3. Did you behave? (Auditing — sign-in logs, risk detections, PIM)

Every Azure subscription is linked to an Entra ID tenant. A tenant is your organization's private directory of users, groups, applications, and policies. When PostureIQ scans "Entra," it's reading your tenant's configuration via the Microsoft Graph API.

Microsoft Graph is the unified API for all Microsoft 365 services. Instead of one API for users, another for groups, and another for policies, Graph provides a single endpoint: https://graph.microsoft.com.

Key Entra ID Concepts

👤 Users & Groups

Users are identities — employees, contractors, guests. Groups organize users for bulk permission assignment. Key attributes: member vs. guest, synced vs. cloud-only, enabled vs. disabled, last sign-in date.

🔐 Multi-Factor Authentication (MFA)

Requiring a second proof of identity beyond a password: a phone notification, a hardware key (FIDO2), or biometrics. MFA blocks 99.9% of credential attacks. PostureIQ checks whether MFA is required for all users and whether admins use strong (phishing-resistant) methods.

🚦 Conditional Access (CA)

Rules that evaluate sign-in context: "If user is an admin AND not on a trusted network, THEN require MFA + compliant device." CA policies are the backbone of Zero Trust. PostureIQ collects every CA policy and evaluates coverage.

👑 Privileged Identity Management (PIM)

Instead of permanent admin roles, PIM grants time-limited, approval-required elevation. Example: "I need Global Admin for 2 hours to fix something." PostureIQ checks PIM configuration: activation duration, MFA requirement, approvals.

📋 Access Reviews

Periodic certifications where managers confirm "yes, this person still needs this access." PostureIQ checks whether access reviews are configured for privileged roles.

🏗️ App Registrations & Service Principals

Applications that authenticate against Entra ID have app registrations (the definition) and service principals (the instance). PostureIQ audits credential expiry, multi-tenant exposure, and excessive OAuth consent grants.

FedRAMP — Federal Risk and Authorization Management Program

Required for any cloud service used by US federal agencies. FedRAMP defines three impact levels: Low (100+ controls), Moderate (325+ controls), and High (421+ controls). PostureIQ maps 38 FedRAMP controls that can be automatically verified via Azure and Entra data.

Key areas: AC (Access Control), AU (Audit & Accountability), IA (Identification & Authentication), SC (System & Communications Protection), SI (System & Information Integrity).

NIST 800-53 Rev 5 — The "Master List"

NIST Special Publication 800-53 is the most comprehensive US security control catalog. FedRAMP inherits from NIST 800-53. PostureIQ maps 27 NIST controls across access control, audit, identification, system protection, and risk assessment families.

CIS Azure Foundations Benchmark v2.0

The Center for Internet Security publishes community-developed benchmarks — step-by-step hardening guides. CIS Azure covers identity (Section 1), Defender (Section 2), storage (Section 3), database (Section 4), logging (Section 5), networking (Section 6), VMs (Section 7), key management (Section 8). PostureIQ maps 24 CIS controls.

ISO/IEC 27001:2022

The international gold standard for information security management systems (ISMS). Annex A defines 93 controls in 4 themes: Organizational, People, Physical, Technological. PostureIQ maps 24 ISO 27001 Annex A controls to automated checks.

PCI DSS v4.0 — Payment Card Industry

Mandatory for any organization that stores, processes, or transmits credit card data. 12 requirements covering network security, data protection, access control, monitoring, and testing. PostureIQ maps 24 PCI-DSS controls.

MCSB — Microsoft Cloud Security Benchmark

Microsoft's own set of best practices for Azure. Organized by security domain (Network Security, Identity Management, Privileged Access, Data Protection, Asset Management, Logging & Threat Detection, Incident Response, Posture & Vulnerability Management, Endpoint Security, Backup & Recovery, DevOps Security, Governance & Strategy). PostureIQ maps 26 MCSB controls.

What Is "Evidence"?

In compliance, evidence is proof that a control is implemented. Examples:

  • "Here's a screenshot showing MFA is enabled" → old way
  • "Here's a JSON record from the Graph API showing 95% of users have registered MFA" → PostureIQ way

PostureIQ collects evidence programmatically — it calls Azure and Graph APIs, normalizes the responses into standardized evidence records, and stores them in JSON. Each record has a unique ID, timestamp, source, and structured data.

Part 4: PostureIQ — What, Why, and How L200

PostureIQ is an open-source AI-powered compliance assessment agent for Microsoft Azure and Entra ID. It connects to your tenant, collects evidence from 64 async collectors (44 Azure + 20 Entra, producing 218 evidence types), evaluates it against 11 compliance frameworks using 10 domain evaluators with 113 check functions, and produces professional reports via 28 report generators (HTML, Markdown, Excel, CSV, JSON, SARIF, OSCAL) — all read-only, all automated, typically in under 5 minutes.

Core Principles

🔒 Read-Only

The agent never creates, modifies, or deletes anything. All Azure SDK calls are list() and get(). All Graph calls are GET. Safe to run on production tenants.

⚡ Async & Parallel

Built on Python asyncio. Azure collectors run in batches of 4, Entra collectors in batches of 3. A full assessment of a large tenant completes in minutes, not hours.

🧩 Multi-Framework

One scan, multiple reports. Collect evidence once, evaluate against FedRAMP + CIS + ISO 27001 simultaneously. Each framework has its own JSON mapping file — easy to customize or add new frameworks.

🤖 AI-Native

Designed as an AI Foundry agent with tool-calling capabilities. Can be deployed to Azure AI Foundry and invoked by enterprise AI platforms. Also runs standalone from the command line.

What the Agent Does NOT Do

  • Does not read secret values from Key Vault — only metadata (expiry dates, types)
  • Does not export personal user data — only aggregated counts and rates
  • Does not remediate findings — it reports, you decide
  • Does not replace a 3PAO/auditor — it provides automated evidence to support human audits

Glossary L100

ARM
Azure Resource Manager — the control plane API for managing Azure resources.
CA
Conditional Access — Entra ID policies that enforce security decisions based on sign-in context.
CIS
Center for Internet Security — non-profit publishing community-built security benchmarks.
CMK
Customer-Managed Key — encryption keys you control (vs. platform-managed).
Control
A specific security requirement from a compliance framework.
DDoS
Distributed Denial of Service — overwhelming a system with traffic to make it unavailable.
Entra ID
Microsoft's cloud identity service (formerly Azure AD). Manages users, auth, and access.
Evidence
Proof that a compliance control is implemented — e.g., a JSON record from an API call.
FedRAMP
Federal Risk and Authorization Management Program — US government cloud security authorization.
FIDO2
Fast Identity Online 2 — passwordless, phishing-resistant authentication standard.
Finding
A compliance assessment result — compliant, non-compliant, or partial.
Framework
A structured set of security controls (e.g., FedRAMP, ISO 27001, PCI-DSS).
Graph API
Microsoft Graph — unified REST API for Microsoft 365 and Entra ID data.
ISO 27001
International standard for information security management systems (ISMS).
MDE
Microsoft Defender for Endpoint — endpoint detection and response (EDR) agent.
MFA
Multi-Factor Authentication — requiring two or more identity proofs.
MCSB
Microsoft Cloud Security Benchmark — Microsoft's Azure best-practice framework.
NIST
National Institute of Standards and Technology — US standards body for security.
NSG
Network Security Group — Azure firewall rules for VNets and subnets.
PCI-DSS
Payment Card Industry Data Security Standard — required for credit card processing.
PIM
Privileged Identity Management — just-in-time, time-limited role elevation.
RBAC
Role-Based Access Control — permissions assigned through roles, not directly.
SSPR
Self-Service Password Reset — users reset their own passwords securely.
TDE
Transparent Data Encryption — automatic encryption for Azure SQL databases.
TLS
Transport Layer Security — encrypts data in transit (e.g., HTTPS).
Tenant
Your organization's isolated Entra ID directory of users, apps, and policies.
VNet
Virtual Network — a private, isolated network in Azure.
3PAO
Third-Party Assessment Organization — auditors who verify FedRAMP compliance.
Zero Trust
"Never trust, always verify" — every access request is authenticated and authorized.
CSA CCM
Cloud Security Alliance Cloud Controls Matrix — security controls for cloud computing across 17 domains.
DLP
Data Loss Prevention — policies and tools that prevent unauthorized data exfiltration.
Drift
Configuration drift — unplanned changes to resource settings that may degrade compliance posture.
GDPR
General Data Protection Regulation — EU regulation governing personal data privacy and protection.
HIPAA
Health Insurance Portability and Accountability Act — US law protecting health information privacy and security.
NIST CSF
NIST Cybersecurity Framework — voluntary guidance for managing cybersecurity risk across 5 core functions.
OSCAL
Open Security Controls Assessment Language — NIST standard for machine-readable compliance documentation.
SARIF
Static Analysis Results Interchange Format — OASIS standard for tool output, used by GitHub and Azure DevOps.
SIEM
Security Information and Event Management — centralized log analysis and security monitoring platform.
SOC 2
System and Organization Controls 2 — AICPA audit standard for service organization security, availability, and privacy.
Suppression
A rule that marks a known-acceptable finding as suppressed — excluded from compliance scoring.
