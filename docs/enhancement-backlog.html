✅ v45 — Multi-Hop Attack Path Analysis
Deepened PostureIQ attack_paths.py with 5 new multi-hop detection categories using existing collected evidence.
Implementation Details ✅ Shipped 2026-04-15

Zero new collectors required — all detections use existing PostureIQ evidence index.

HIGH Key Vault → Identity → Resource

Principals with Key Vault read access AND privileged roles elsewhere. Compromising this identity exposes vault secrets AND privileged resource access.

Evidence: azure-keyvault, azure-role-assignment | Score: 88

HIGH App/Function → MI → Resource

Web Apps or Functions with system-assigned Managed Identity holding privileged roles. Exploiting the app via SSRF/RCE grants privileged Azure access via the MI token endpoint.

Evidence: azure-webapp-config, azure-function-app, azure-role-assignment | Score: 87

CRITICAL Conditional Access Bypass

Privileged Entra roles (Global Admin, Security Admin, etc.) without MFA enforcement via Conditional Access. Stolen credentials provide direct privileged access.

Evidence: entra-conditional-access-policy, entra-role-assignment | Score: 92

MEDIUM–HIGH Weak Credential + Privileged SP

Service Principals with expired or expiring credentials that hold privileged Azure roles. Creates a window for unauthorized access or automation failure.

Evidence: entra-application, entra-service-principal, azure-role-assignment | Score: 75–82

CRITICAL Network Pivot — Internet-Exposed VM

VMs in subscriptions with Internet-exposed NSG rules AND privileged Managed Identity. Attacker can request MI token from IMDS (169.254.169.254) and pivot to privileged Azure access.

Evidence: azure-nsg-rule, azure-vm-config, azure-role-assignment | Score: 93

New Detection Capabilities (beyond v45)

Multi-Hop Traversal (3+ hops)

A → B → C → D chains where no single hop is critical but the chain creates a critical path.

Cross-Subscription Pivots

Identity in subscription A with role in subscription B that trusts subscription C.

VNet Peering Chains

VNet A ↔ VNet B ↔ VNet C creating network adjacency across subscriptions.

Private Endpoint Bypass

Resource with private endpoint but identity has both network AND data-plane access.

Full 4-Hop Identity Chain

App Registration → Service Principal → Managed Identity → Resource.

LLM Attacker Reasoning

"Given this graph, what would an APT actor target first?" — emergent pattern discovery.

Cross-Cloud Attack Patterns

CRITICAL S3 Credential Leak → Azure Pivot

AWS S3 bucket with public access contains Azure SP credentials → full Azure subscription access.

CRITICAL Azure AD Trust → AWS AssumeRole

Azure AD identity trusted by AWS IAM role → cross-cloud lateral movement.

HIGH GCP Workload Identity → Azure MI

GCP service linked to Azure Managed Identity → cloud-to-cloud pivot.

HIGH Shared VPN/ExpressRoute

Network paths connecting cloud VPCs/VNets enabling lateral movement.

HIGH Container Registry Poisoning

ACR/ECR/GCR image shared across clouds → supply chain attack path.

HIGH Shared Secrets Across Vaults

Same credentials stored in Key Vault, Secrets Manager, and Secret Manager.
