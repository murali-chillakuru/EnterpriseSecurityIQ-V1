Overview

The AI Agent Security Engine (ai_agent_security_engine.py, 6,611 lines) performs a comprehensive, evidence-driven security assessment of AI agents across 6 distinct platforms. It evaluates Copilot Studio bots, Microsoft Foundry workspaces and agent applications, Entra ID identity configurations, custom agent deployments, AI infrastructure resources, and agent orchestration patterns. The engine collects evidence from 3 dedicated collectors, synthesizes cross-platform telemetry, runs 50+ category-level analyzers producing 120+ individual checks, and generates severity-scored findings with actionable remediation guidance. All operations are strictly read-only — no tenant resources are modified.

⚡

Assessment Pipeline

The pipeline flows from CLI invocation through evidence collection, cross-platform synthesis, platform-specific analysis, scoring, and multi-format report generation.

  1. CLI invocation — run_ai_agent_security.py parses arguments and authenticates credentials.
  2. Evidence collection — Three collectors gather Copilot Studio bots/DLP/environments, Foundry workspaces/deployments/connections, and Entra AI service principals/consent grants.
  3. Cross-platform synthesis — _synthesize_agent_orchestration_config() merges evidence into unified agent orchestration telemetry.
  4. Category analysis — 50+ analyzers run 120+ individual checks across all 6 platforms.
  5. Scoring — Severity-weighted scoring with platform and compliance breakdowns.
  6. Report generation — HTML dashboard, Excel workbook, CSV flat file, PDF, and remediation scripts.

Scoring Model

Findings are scored using severity weights, aggregated per-category and overall, then mapped to risk levels. The engine also generates PlatformBreakdown and ComplianceBreakdown summaries.

Severity Weights

SeverityWeightBadge
Critical10.0critical
High7.5high
Medium5.0medium
Low2.5low
Informational1.0info
Score formula: Per-category risk = min(100, sum(severity_weights) × 5). Overall score = weighted average by finding count across all categories. A lower score is better.

Risk Levels

Score RangeRisk Level
≥ 75Critical
≥ 50High
≥ 25Medium
< 25Secure
Breakdowns: The engine produces a PlatformBreakdown (score per platform A–F) and a ComplianceBreakdown (findings mapped to compliance framework controls) as part of the assessment result.

CLI Usage

The runner script provides flexible CLI options for full assessments, evidence reuse, category targeting, trend comparison, and CI/CD gating.

# Full assessment
python run_ai_agent_security.py --tenant <tenant-id>

# Reuse existing evidence
python run_ai_agent_security.py --tenant <tenant-id> --evidence raw-evidence.json

# Target specific categories
python run_ai_agent_security.py --tenant <tenant-id> --category cs_authentication,foundry_network

# Trend comparison against a previous run
python run_ai_agent_security.py --tenant <tenant-id> --previous-run prior-assessment.json

# CI/CD gate — fail pipeline on critical findings
python run_ai_agent_security.py --tenant <tenant-id> --fail-on-severity critical
ArgumentDescription
--tenantAzure AD tenant ID (required)
--evidencePath to pre-collected evidence JSON to reuse
--categoryComma-separated category prefixes to target
--previous-runPath to a prior assessment JSON for trend delta
--fail-on-severityExit non-zero if findings at this severity or above exist
--output-dirDirectory for output artifacts (default: output/)
--auth-modeAuthentication mode: auto, azurecli, serviceprincipal, appregistration
--formatOutput formats: html, excel, csv, pdf, json, all
