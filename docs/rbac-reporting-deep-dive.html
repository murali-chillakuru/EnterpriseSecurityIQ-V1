Audience: IAM teams, auditors · Prerequisites: Architecture · Related: Agent Capabilities · Evaluation Rules
🔐

Overview

The RBAC Reporting module provides a complete audit of Azure role-based access control across an entire management-group hierarchy. It collects every role assignment — including PIM-eligible — at management-group, subscription, resource-group, and resource scopes, resolves all principal identities through Microsoft Graph (batch getByIds + service principal enrichment), expands group memberships transitively, detects 6 categories of privilege risks, computes a 0–100 RBAC security score, and produces both an interactive HTML report with SVG charts and an 8-tab Excel workbook for offline analysis.

⚡

Pipeline

The 7-phase collection pipeline orchestrates calls across four Azure APIs — Management Groups, Authorization, Resource Manager, and Microsoft Graph — before handing the enriched dataset to the report generator.

7-Phase RBAC Collection Pipeline
sequenceDiagram participant R as run_rbac_report.py participant C as rbac_collector.py participant MG as Management Groups API participant AUTH as Authorization API participant ARM as Resource Manager API participant G as Microsoft Graph participant RPT as rbac_report.py R->>C: collect_rbac_data(creds, subscriptions) C->>MG: Phase 1 → Build MG tree MG-->>C: Hierarchical tree C->>AUTH: Phase 2 → MG-scope assignments + PIM AUTH-->>C: MG assignments C->>AUTH: Phase 3 → Per-sub assignments C->>ARM: Phase 3 → RGs + resources AUTH-->>C: Sub/RG/resource assignments C->>C: Phase 4 → Attach subs to MG tree C->>G: Phase 5 → Resolve principals (batch) G-->>C: Names + types C->>G: Phase 6 → Expand groups (transitive) G-->>C: Group members C->>C: Phase 7 → Stats + risks + score C-->>R: {tree, principals, stats, risks, score} R->>RPT: generate_rbac_report() RPT-->>R: HTML + Excel
  1. Phase 1 — Build the management-group tree by recursively expanding children.
  2. Phase 2 — Collect role assignments + PIM eligibility at every MG scope.
  3. Phase 3 — For each subscription: list assignments, resource groups, and resources.
  4. Phase 4 — Attach subscription nodes into the MG tree structure.
  5. Phase 5 — Batch-resolve all principal IDs to display names and types via Graph.
  6. Phase 6 — Expand group memberships transitively (2-level nesting).
  7. Phase 7 — Compute aggregate stats, detect risks, calculate RBAC score.

Risk Detection

After collection and identity resolution, the engine evaluates every assignment against 6 risk rules, each producing findings with severity, scope, and remediation guidance.

#SeverityCondition
1CriticalStanding Owner or User Access Administrator at Management Group / Subscription scope
2HighStanding Contributor at Management Group scope
3HighService Principal with Owner role (any scope)
4HighGroup with >20 members holding a privileged role
5MediumCustom role at Management Group or Subscription scope
6MediumService Principal with Contributor at Management Group / Subscription scope
📊

RBAC Score Computation

The RBAC Security Score starts at 100 and applies deductions based on risk findings and ratio analysis, clamped to [0, 100].

DeductionPoints
Each Critical risk finding−15
Each High risk finding−8
Each Medium risk finding−4
Privileged-to-total ratio > 50%−15
Privileged-to-total ratio > 30%−8
Zero PIM eligible when privileged active > 0−10
Formula: score = max(0, min(100, 100 − Σ(deductions))) — The score is clamped to the [0, 100] range after all deductions are applied.

CLI Usage

The run_rbac_report.py entry point supports tenant-scoped or subscription-scoped runs with configurable output directories. 

# Full RBAC report for a tenant
python run_rbac_report.py --tenant <tenant-id>

# Specific subscriptions only
python run_rbac_report.py --tenant <tenant-id> --subscriptions sub-id-1,sub-id-2

# Custom output directory
python run_rbac_report.py --tenant <tenant-id> --output-dir ./my-output
ArgumentRequiredDescription
--tenantYesAzure AD tenant ID
--subscriptionsNoComma-separated subscription IDs (default: all accessible)
--output-dirNoOutput directory (default: output/<timestamp>)
