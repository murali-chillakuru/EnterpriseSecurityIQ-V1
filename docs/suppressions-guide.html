🔇 Overview

The suppression system lives in AIAgent/app/postureiq_evaluators/suppressions.py and exposes two functions:

FunctionSignatureReturns
load_suppressions(path: str) → list[dict]Parsed rules array; [] on missing file or parse error
apply_suppressions(findings, rules) → tuple(active_findings, suppressed_findings)
💡 Two Input Formats Accepted load_suppressions accepts both bare arrays ([{...}]) and envelope format ({"suppressions": [{...}]}).

📄 File Format

Create suppressions.json in your working directory or pass a custom path via --suppressions.

[
  {
    "control_id": "AC-6",
    "resource": ".*",
    "reason": "Accepted risk — JIT compensating control",
    "expires": "2026-12-31"
  },
  {
    "control_id": "SC-28.*",
    "resource": ".*temp-storage.*",
    "reason": "Temporary storage — no sensitive data",
    "expires": "2026-06-30"
  },
  {
    "control_id": "IA-5",
    "reason": "Permanent exception — federated auth"
  }
]

📋 Rule Fields

FieldRequiredTypeMatchingDescription
control_idYesstringre.fullmatch()Control ID pattern — exact or regex
resourceNostringre.search() case-insensitiveResource identifier regex. Default: .*
reasonYesstring—Human-readable justification
expiresNostringISO YYYY-MM-DDExpiry date. Omit for permanent
⚠️ fullmatch vs search The control_id uses re.fullmatch() — the entire control ID must match. Use AC-6.* to match AC-6 and AC-6.1. The resource field uses re.search() — partial matches work.

🎯 Matching Logic

Each finding is checked against rules in array order. First matching rule wins.

1

Check Expiry

If expires is set and the date has passed, the rule is skipped. Invalid date formats are also skipped with a WARNING log.

2

Match control_id (Two-Stage)

Stage A: Exact string equality (rule.control_id == finding.ControlId).
Stage B: If no exact match → re.fullmatch(rule.control_id, finding.ControlId).

3

Match resource

If the rule has a resource field: re.search(rule.resource, finding.ResourceId, re.IGNORECASE). If no resource field, matches all resources.

4

Both Must Pass

A finding is suppressed only when both control_id and resource match (or resource is absent). Suppression fields are injected into the finding dict.

⏰ Expiry Handling

ScenarioBehaviourLog Level
expires in the pastRule skippedWARNING
expires within 7 daysRule applied, warning loggedWARNING
expires in the futureRule applied normally—
expires missingPermanent — applies indefinitely—
Invalid date formatRule skippedWARNING

📝 Audit Trail

When a finding is suppressed, three fields are dynamically injected into the finding dict:

FieldTypeDescription
suppressed_reasonstringThe reason from the matching rule
suppressed_rule_indexint1-based index of the matching rule
suppressed_expiresstring / nullThe expires date from the rule
✅ Full Audit Visibility

Active findings → compliance score + reports. Suppressed findings → JSON data exports for review. These fields are NOT part of the FindingRecord dataclass — injected at runtime.

📚 Pattern Examples

Single Control

Suppress one control on all resources:

{"control_id":"AC-6","reason":"JIT compensating control"}

Control Family

Suppress all SC-28.x controls:

{"control_id":"SC-28.*","reason":"Family exempted"}

Resource Pattern

Suppress non-prod resources:

{"control_id":".*","resource":".*-dev-.*|.*-test-.*","reason":"Non-prod","expires":"2026-12-31"}

Temporary

7-day exception with tracking:

{"control_id":"IA-2","reason":"MFA rollout — ESIQ-1234","expires":"2026-04-18"}

Framework-Specific

Prefix-aware suppression:

{"control_id":"NIST-AC-6","reason":"NIST-specific exception"}

Escaped Dots

PCI controls with literal dots:

{"control_id":"PCI-8\\.3\\..*","reason":"SSO handles auth"}

Review Workflow

1

Initial Assessment

Run full assessment and review all findings.

2

Identify Accepted Risks

Tag false positives and compensating controls.

3

Create Rules

Write suppressions.json with specific patterns and expiry dates.

4

Re-Run

Suppressed findings excluded from score, but present in JSON exports.

5

Monitor Expiry

Pipeline logs warn about rules expiring within 7 days.

6

Clean Up

Remove or update expired rules periodically.
