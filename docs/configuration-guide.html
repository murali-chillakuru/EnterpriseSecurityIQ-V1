Configuration Loading

The main configuration file is config/enterprisesecurityiq.config.json. Copy examples/sample-config.json as a starting point.

sequenceDiagram participant U as User / CI participant ENV as Environment Vars participant CFG as Config JSON File participant PY as config.py participant AC as AssessmentConfig U->>ENV: Set ENTERPRISESECURITYIQ_CONFIG (optional) U->>ENV: Set AZURE_TENANT_ID, etc. (optional) PY->>PY: from_env() called alt ENTERPRISESECURITYIQ_CONFIG is set PY->>CFG: from_file(path) CFG-->>PY: JSON values (camelCase → snake_case) else No config file PY->>PY: Use dataclass defaults end PY->>ENV: Check override env vars ENV-->>PY: Override file/default values PY->>AC: Final AssessmentConfig Note over AC: Thresholds loaded via
field iteration (snake_case keys)

Loading Sequence Explained

  1. from_env() checks if ENTERPRISESECURITYIQ_CONFIG is set — if so, calls from_file() first
  2. from_file() reads JSON, maps camelCase keys to snake_case dataclass fields
  3. Thresholds are loaded dynamically by iterating ThresholdConfig.__dataclass_fields__
  4. Environment variables override file-loaded values (highest priority)
  5. Any unset fields use Python dataclass defaults

Precedence Order

1
Environment Variables — Override everything
AZURE_TENANT_ID, ENTERPRISESECURITYIQ_FRAMEWORKS, etc.
2
Config JSON File — from_file() values
config/enterprisesecurityiq.config.json
3
Python Dataclass Defaults — Code-level defaults
AssessmentConfig, AuthConfig, CollectorConfig, ThresholdConfig

Auth Modes

auto (default)
DefaultAzureCredential
Chains: env vars → managed identity → Azure CLI → VS Code
Best for: Local dev, Azure-hosted workloads
serviceprincipal
ClientSecretCredential
Requires: AZURE_CLIENT_ID + AZURE_CLIENT_SECRET
Best for: CI/CD pipelines
appregistration
ClientSecretCredential
Uses: ENTERPRISESECURITYIQ_APP_CLIENT_ID (falls back to AZURE_CLIENT_ID)
Best for: Dedicated app registration
azurecli
AzureCliCredential
Requires: Active az login session
Best for: Force CLI-only auth

Preflight Check

ComplianceCredentials.preflight_check() validates access before collection:

  • ARM: Subscription listing
  • Graph (Users): User.Read.All
  • Graph (CA): Policy.Read.All
  • Graph (Roles): RoleManagement.Read.All

403 responses → warnings (collection continues with reduced scope).
401 responses → blocking errors.

Returns: ok, user, tenant, roles, arm_subs, graph_ok, warnings, errors

Running an Assessment

Prerequisites

  1. Python 3.10+ (3.12 recommended)
  2. Install dependencies: cd AIAgent && pip install -r requirements.txt
  3. Authenticate: az login, environment variables, or managed identity

CLI Commands

# Interactive (prompts for tenant)
python run_copilot_readiness.py

# CLI arguments
python run_copilot_readiness.py --tenant "4a3eb5f4-..." --fail-on-severity high

# Using config file
ENTERPRISESECURITYIQ_CONFIG=../config/enterprisesecurityiq.config.json python run_data_security.py
