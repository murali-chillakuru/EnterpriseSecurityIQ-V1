RBAC Roles

Any one: Global Admin, Global Reader, Security Admin/Reader, Compliance Admin.

🔐 Authentication Modes

Set via ENTERPRISESECURITYIQ_AUTH_MODE env var or auth_mode in config.

auto Default

DefaultAzureCredential — chains az CLI, managed identity, OIDC, env vars. Best for GitHub Actions OIDC and Azure Managed Identity.

serviceprincipal

ClientSecretCredential — requires AZURE_CLIENT_ID, AZURE_CLIENT_SECRET, AZURE_TENANT_ID. Traditional CI/CD pattern.

appregistration

ClientSecretCredential — uses ENTERPRISESECURITYIQ_APP_CLIENT_ID / APP_CLIENT_SECRET (falls back to AZURE_ equivalents).

azurecli

AzureCliCredential — uses active az login session. For local dev and manual pipeline runs.

🐙 GitHub Actions

Full Assessment with OIDC + Compliance Gate 
name: Compliance Assessment
on:
  schedule:
    - cron: '0 6 * * 1'  # Weekly Monday 6am UTC
  workflow_dispatch:

permissions:
  id-token: write
  contents: read

jobs:
  assess:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - name: Install dependencies
        run: pip install -r AIAgent/requirements.txt

      - name: Azure Login (OIDC)
        uses: azure/login@v2
        with:
          client-id: ${{ secrets.AZURE_CLIENT_ID }}
          tenant-id: ${{ secrets.AZURE_TENANT_ID }}
          subscription-id: ${{ secrets.AZURE_SUBSCRIPTION_ID }}

      - name: Run Copilot Readiness Assessment
        working-directory: AIAgent
        run: python run_copilot_readiness.py --tenant ${{ secrets.AZURE_TENANT_ID }} --fail-on-severity high
        env:
          AZURE_TENANT_ID: ${{ secrets.AZURE_TENANT_ID }}
          ENTERPRISESECURITYIQ_AUTH_MODE: auto

      - name: Compliance Gate
        run: |
          OUTDIR=$(ls -td AIAgent/output/*/ | head -1)
          SCORE=$(python -c "
          import json, glob, pathlib
          files = glob.glob(str(pathlib.Path('${OUTDIR}') / 'data_exports' / '*.json'))
          r = json.load(open(files[0])) if files else {}
          print(r.get('CompliancePercentage', 0))
          ")
          echo "Compliance Score: $SCORE%"
          if (( $(echo "$SCORE < 70" | bc -l) )); then
            echo "::error::Compliance score $SCORE% below 70%"
            exit 1
          fi

      - uses: actions/upload-artifact@v4
        with:
          name: compliance-report-${{ github.run_number }}
          path: AIAgent/output/
          retention-days: 90
Specialized Assessment with --fail-on-severity 
      - name: Data Security Assessment
        working-directory: AIAgent
        run: |
          python run_data_security.py \
            --tenant ${{ secrets.AZURE_TENANT_ID }} \
            --fail-on-severity critical \
            --quiet
        # Exit code 1 if any critical-severity finding exists

🔷 Azure DevOps Pipeline

Full Pipeline — Assessment + Copilot Readiness 
trigger: none
schedules:
  - cron: '0 6 * * 1'
    displayName: Weekly Compliance Check
    branches:
      include: [main]

pool:
  vmImage: 'ubuntu-latest'

variables:
  - group: EnterpriseSecurityIQ-Secrets

steps:
  - task: UsePythonVersion@0
    inputs:
      versionSpec: '3.12'

  - script: pip install -r AIAgent/requirements.txt
    displayName: Install Dependencies

  - task: AzureCLI@2
    displayName: Run Copilot Readiness Assessment
    inputs:
      azureSubscription: 'EnterpriseSecurityIQ-SPN'
      scriptType: bash
      scriptLocation: inlineScript
      inlineScript: |
        cd AIAgent
        export ENTERPRISESECURITYIQ_AUTH_MODE=auto
        python run_copilot_readiness.py --tenant $(AZURE_TENANT_ID) --fail-on-severity high

  - task: AzureCLI@2
    displayName: Copilot Readiness Gate
    inputs:
      azureSubscription: 'EnterpriseSecurityIQ-SPN'
      scriptType: bash
      scriptLocation: inlineScript
      inlineScript: |
        cd AIAgent
        python run_copilot_readiness.py \
          --tenant $(AZURE_TENANT_ID) \
          --fail-on-severity high

  - publish: $(System.DefaultWorkingDirectory)/AIAgent/output
    artifact: compliance-report
    displayName: Publish Reports

🚦 Compliance Gate Strategy

90–100% PASS

No action needed. Pipeline completes successfully.

70–89% WARN

Create tracking ticket. Pipeline continues with warning annotation.

< 70% FAIL

Block deployment. Pipeline exits with code 1.

💡 --fail-on-severity For specialized assessments (run_data_security.py, run_copilot_readiness.py, run_ai_agent_security.py), use --fail-on-severity <level>. Exits with code 1 if any finding meets or exceeds that severity.

🔁 Determinism Checks

Verify that evaluation is reproducible given identical evidence. Run in CI/CD to catch non-deterministic regressions.

sequenceDiagram participant CI as CI Pipeline participant S as Determinism Script participant C as Collectors participant E as Evaluators participant H as SHA-256 Hasher CI->>S: Launch determinism check S->>C: Phase A: Collect evidence (once) C-->>S: Evidence snapshot loop Phase B: N identical-order runs S->>E: Evaluate all controls E-->>S: Findings + scores end loop Phase C: N shuffled-order runs S->>E: Evaluate (shuffled seed: i*42+7) E-->>S: Findings + scores end S->>H: Strip volatile fields → SHA-256 H-->>S: All hashes match? PASS / FAIL
Step 1 of 6
Launch Check

CI pipeline triggers the determinism script with --tenant and optionally --runs N (default 3).

CI Pipeline → Determinism Script
Step 2 of 6
Phase A: Collect Once

Evidence is collected exactly once. All subsequent evaluation runs reuse this identical evidence snapshot.

Script → Collectors
Step 3 of 6
Phase B: Identical Order

N evaluation passes run with controls in their original order. Each produces findings, scores, and a summary.

Script → Evaluators (N times)
Step 4 of 6
Phase C: Shuffled Order

N more passes with controls shuffled using seed i * 42 + 7. Ensures order-independence.

Script → Evaluators (N times, shuffled)
Step 5 of 6
Phase D: Hash Comparison

Volatile fields (FindingId, EvaluatedAt, Timestamp, etc.) stripped. SHA-256 computed for summary, control_results, findings, missing_evidence.

Script → SHA-256 Hasher
Step 6 of 6
Verdict

If all hashes match across all runs: exit 0 (PASS). Any mismatch: exit 1 (FAIL). Phase E also checks report generation determinism.

Hasher → Script → CI Pipeline

Volatile Fields Stripped Before Comparison

FindingId, EvaluatedAt, RecordId, RecordedAt, EvidenceId, CollectedAt, AssessmentId, StartedAt, CompletedAt, DurationSeconds, OutputDirectory, Timestamp

📊 Delta Tracking

📈 Automatic Drift Detection When .last_run.json exists (saved after every run), the next assessment generates a drift report showing: new findings, resolved findings, and score change (+/−%).

Store output directories as pipeline artifacts to maintain history. The drift_html/ subfolder appears in the output only when prior-run data is available.

Checkpoint / Resume

Collection uses .checkpoint.json for fault tolerance:

  • Stores {completed, failed, evidence} during collection
  • On resume, completed collectors are skipped
  • Corrupt checkpoint file → fresh start
  • Auto-deleted after successful collection
