Findings — Detailed

HIGH H-1: Container App publicly exposed without IP restrictions
The Container App has external: true ingress with no ipSecurityRestrictions. Anyone on the internet can reach the /chat and / endpoints. While the app requires Entra ID authentication, the attack surface is unnecessarily large.
Remediation: Add IP security restrictions to allow only your corporate egress IPs, or deploy behind Azure Front Door with WAF. For VNet isolation, deploy the Container Apps Environment into a VNet with internal-only ingress.
az containerapp ingress access-restriction set -n esiqnew-agent -g ESIQNew-RG --rule-name corp-only --ip-address <CIDR> --action Allow
HIGH H-2: Container Apps Environment has no VNet integration
The environment is deployed without a VNet (vnetConfiguration: null). All inter-service traffic traverses public endpoints. This means the Container App → Azure OpenAI, Storage, and other calls go over the public internet, even though they use managed identity RBAC.
Remediation: Create a VNet-integrated Container Apps Environment with private endpoints for dependent services. This requires rebuilding the environment (cannot retrofit). Use az containerapp env create --infrastructure-subnet-resource-id. Add private endpoints for Storage, Key Vault, AI Services, and ACR.
HIGH H-3: Storage account allows traffic from all networks
Storage account esiqnewstorage has networkRuleSet.defaultAction: Allow, meaning any Azure service or internet client with valid credentials can access blobs. While blob public access is disabled and shared key is off (enforcing Entra ID auth), network-layer restriction adds defense in depth.
Remediation: Set default action to Deny and add exceptions for the Container Apps subnet (requires VNet integration).
az storage account update -n esiqnewstorage -g ESIQNew-RG --default-action Deny
Then add VNet rules or private endpoint.
MEDIUM M-1: Azure OpenAI / AI Services has public network access enabled
ESIQNew-AI has publicNetworkAccess: Enabled. Although local auth is disabled (requiring Entra ID tokens), the endpoint is reachable from the public internet.
Remediation: Disable public access and configure a private endpoint. Requires VNet-integrated Container Apps Environment.
az cognitiveservices account update -n ESIQNew-AI -g ESIQNew-RG --public-network-access Disabled
MEDIUM M-2: ACR has public network access and Basic SKU
esiqnewacr is Basic SKU with public access enabled. Basic SKU does not support private endpoints, content trust, or customer-managed encryption. Admin user is correctly disabled.
Remediation: Upgrade to Standard or Premium SKU to enable private endpoints and content trust.
az acr update -n esiqnewacr -g ESIQNew-RG --sku Premium --public-network-enabled false
MEDIUM M-3: Key Vault lacks purge protection
ESIQNew-kv has soft delete enabled but purge protection is not enabled. A privileged user could permanently delete secrets/keys during the soft-delete retention window.
Remediation: Enable purge protection (irreversible — cannot be disabled once set).
az keyvault update -n ESIQNew-kv -g ESIQNew-RG --enable-purge-protection true
MEDIUM M-4: No zone redundancy on Container Apps Environment
Environment has zoneRedundant: false. A single availability zone failure would take down the agent.
Remediation: Recreate the environment with --zone-redundant flag. Cannot be changed after creation — requires redeployment.
MEDIUM M-5: No CORS policy on Container App ingress
The Container App has no CORS policy configured (corsPolicy: null). If the SPA is served from the same origin this is acceptable, but if ever served from a different domain, the lack of CORS restrictions could be exploited.
Remediation: Set explicit CORS allowed origins matching your SPA domain.
az containerapp ingress cors update -n esiqnew-agent -g ESIQNew-RG --allowed-origins "https://esiqnew-agent.delightfulbush-cc9cf399.northeurope.azurecontainerapps.io"
LOW L-1: Log Analytics has short retention (30 days)
ESIQNew-law retains logs for only 30 days. For security incident investigation and compliance, 90+ day retention is recommended.
Remediation: Increase retention or configure archiving.
az monitor log-analytics workspace update -n ESIQNew-law -g ESIQNew-RG --retention-time 90
LOW L-2: Application Insights local auth not explicitly disabled
ESIQNew-appinsights has disableLocalAuth: null (default allows API key auth). Best practice is to disable local auth and use Entra ID only.
Remediation: Disable local authentication.
az monitor app-insights component update --app ESIQNew-appinsights -g ESIQNew-RG --set properties.disableLocalAuth=true
LOW L-3: Log Analytics and App Insights allow public ingestion
Both monitoring resources have public network access enabled for ingestion and query. In a locked-down environment, telemetry should flow via Azure Monitor Private Link Scope (AMPLS).
Remediation: Configure Azure Monitor Private Link Scope (AMPLS) and set public access to disabled on both resources. Requires VNet integration.

Security Controls Summary

✅ What's Done Well

  • Managed Identity: User-assigned MI with 6 least-privilege role assignments — no shared secrets anywhere
  • Entra ID authentication: User-delegated tokens required for all web requests, MI fallback disabled for web users
  • Storage hardening: Shared key disabled, blob public access off, HTTPS-only, TLS 1.2
  • Key Vault: RBAC authorization, soft delete, public access disabled
  • ACR: Admin user disabled, MI-based image pull
  • AI Services: Local auth disabled, token-based access only
  • Defender for Cloud: All 15 plans at Standard tier — full coverage
  • Logging: Container Apps → Log Analytics, App Insights telemetry enabled
  • Container App: HTTPS-only, no insecure HTTP allowed, no secrets in env vars

⚠️ What Needs Improvement

  • Network isolation: No VNet, no private endpoints — all service-to-service calls over public internet
  • Container App exposure: External ingress with no IP restrictions or WAF
  • Storage network rules: Default action Allow (should be Deny + exceptions)
  • Purge protection: Not enabled on Key Vault
  • Zone redundancy: Not enabled on Container Apps Environment or ACR
  • ACR SKU: Basic — no private endpoints, no content trust
  • Log retention: 30 days on Log Analytics (recommend 90+)
  • Monitoring auth: App Insights local auth not disabled
