Why New Foundry?

Classic vs. New Foundry

The classic deployment used Microsoft.MachineLearningServices/workspaces (ML Hub + ML Project). While functional, it only appeared in the Foundry (classic) portal. The new architecture uses Microsoft.CognitiveServices/accounts with allowProjectManagement and child CognitiveServices/accounts/projects, making it visible in the New Foundry portal.

FeatureClassic (old infra/)New Foundry
Visible in ai.azure.com❌ No✅ Yes
Foundry Resource + Project❌ ML Workspace only✅ CognitiveServices/accounts/projects
Model Management UI❌ Portal only✅ Foundry + Portal

Option A+B: SPA + Foundry Agent

Dual Delivery Model

The deployment now supports a combined Option A+B architecture. A single Container App serves both a SPA web dashboard (Option A, MSAL.js SSO) and registers as a Foundry Agent (Option B, Assistants API) on startup. Both paths share the same 12 agent tools and Azure OpenAI backend.
